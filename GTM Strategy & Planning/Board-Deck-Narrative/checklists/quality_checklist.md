# Quality Checklist

**Purpose of this file:** A technical/process quality checklist for verifying that a Board Deck Narrative output was constructed correctly according to this skill's workflow, validation rules, and scope boundaries.

## Workflow Adherence

- [ ] Decision context (audience, ask, constraints) was clarified before drafting
- [ ] All supplied source material was read and inventoried before drafting began
- [ ] The correct narrative arc (steady progress, course correction, underperformance, pivot, investment ask) was selected and applied
- [ ] The pyramid principle / BLUF discipline was applied (conclusion first, then supporting reasons, then detail)
- [ ] Metrics were selected using leading/lagging/north-star/unit-economics guidance, not simply copied from the source in bulk
- [ ] Strategic priorities were compressed and grouped by outcome/theme, not copy-pasted from the full roadmap
- [ ] Any ask was structured using the options-recommendation-rationale-risk framework
- [ ] The correct template(s) were selected and fully populated (no unfilled bracketed placeholders remain)
- [ ] Output was checked against `checklists/executive_review_checklist.md` prior to delivery

## Scope Compliance

- [ ] The request was confirmed to be narrative condensation/framing, not strategy origination
- [ ] The request was confirmed to be narrative condensation/framing, not budget modeling or financial allocation
- [ ] The request was confirmed to be narrative condensation/framing, not detailed roadmap construction
- [ ] Any out-of-scope portion of a compound request was explicitly flagged and redirected to the appropriate upstream skill
- [ ] No net-new market research, competitive analysis, or financial forecast was fabricated to fill a gap

## Validation Rules Compliance

- [ ] BLUF appears within the first paragraph or slide
- [ ] No more than 5 strategic priorities appear in the output
- [ ] Every metric shown includes context (target, trend, or benchmark)
- [ ] Every risk listed includes a stated business impact
- [ ] Any ask includes at least one alternative option considered
- [ ] Output length respects any stated constraint, or defaults to board-standard brevity if none was stated

## Error Handling Verification

- [ ] Missing metrics/data are marked explicitly (e.g., "[METRIC NOT PROVIDED]") rather than estimated
- [ ] Sparse source material triggered a request for minimum viable input rather than fabricated content
- [ ] Contradictory inputs were surfaced to the user rather than silently resolved
- [ ] Out-of-scope requests were declined with a clear explanation and redirection, not attempted

## Consistency and Reusability

- [ ] No company, product, or brand names appear anywhere in the drafted output template language or supporting notes (only in user-supplied content, which is expected)
- [ ] Terminology is consistent across all sections of the output (summary, metrics, priorities, risks, ask)
- [ ] Structure is consistent with prior cycles' outputs where continuity information was available
- [ ] Language, structure, and framework choices are industry-agnostic and would apply equally across different business types

## Final Delivery Check

- [ ] Output format matches what was requested (Board Summary, Executive Narrative, One-page Strategy, Decision Summary, or Presentation Outline)
- [ ] Any assumptions or gaps are clearly noted at the end of the document
- [ ] The document is ready for direct use with minimal further editing
