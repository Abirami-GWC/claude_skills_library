# Scenario Planning

## Purpose
This document provides standalone reference knowledge on scenario planning and sensitivity analysis for GTM budgets — best/base/worst case modeling and sensitivity testing — so that a budget plan remains resilient under demand, cost, or market uncertainty rather than relying on a single fixed assumption set.

## Introduction
Every budget forecast rests on assumptions (CAC trend, conversion rates, market demand, cost per channel) that carry uncertainty. Scenario planning makes that uncertainty explicit and actionable by modeling multiple plausible futures — rather than presenting one confident number — so that decision-makers can see what changes under different conditions and pre-plan responses (e.g., where to cut first if a worst case materializes, or where to double down if a best case emerges).

## Detailed Explanation

### Best / Base / Worst Case Structure
- **Base case**: the most likely outcome given current trends and stated assumptions; this is the "default" plan used for day-to-day operating decisions.
- **Best case**: an upside scenario reflecting favorable but plausible movement in key drivers (e.g., CAC improves 15%, conversion rate improves, a channel outperforms recent trend). Should be optimistic but grounded — not a best-possible-world fantasy.
- **Worst case**: a downside scenario reflecting unfavorable but plausible movement in key drivers (e.g., CAC rises due to increased competition, a partner relationship weakens, macro conditions soften). Used to pressure-test whether the plan survives adverse conditions and to pre-identify what would be cut first.

Each scenario should vary the same small set of key drivers (typically 3-5: CAC, conversion rate, channel cost inflation, market demand growth) rather than re-deriving the model from scratch, so the three cases remain comparable.

### Sensitivity Analysis
Sensitivity analysis identifies which input variables have the largest effect on the outcome, by varying one input at a time (holding others constant) and observing the effect on the target output (e.g., total customers acquired, total revenue, required budget).
- **Method**: for each key driver, model the outcome at a reasonable "high" and "low" value and compare the swing in outcome; rank drivers by the size of their effect (commonly visualized as a "tornado chart" ranking drivers top to bottom by impact magnitude).
- **Use**: focus data-gathering and risk-mitigation effort on the highest-sensitivity drivers first; low-sensitivity inputs can be estimated more roughly without materially affecting decision quality.

### Scenario Planning for Budget Cuts
When a budget cut is imposed, scenario planning should model the cut not as a uniform percentage reduction across all channels, but as a set of explicit options:
1. **Uniform cut**: reduce every channel by the same percentage — simplest but ignores differences in channel efficiency and protects underperformers.
2. **Efficiency-ranked cut**: cut the lowest-ROI/CAC-payback channels first, protecting the strongest performers as much as possible.
3. **Strategic-protection cut**: protect channels tied to strategic priorities (e.g., a new market entry) even if their near-term ROI is lower, cutting elsewhere to compensate.
Present the resulting revenue/pipeline impact of each option side by side so the trade-off is explicit rather than assumed.

### Sensitivity to Timing
Scenario models should also test timing risk — e.g., what happens if a channel's expected ramp-up takes 2 months longer than planned, or if a key partner deal slips a quarter — since delayed results can be as damaging to a plan as reduced magnitude.

## Professional Guidance
- Always accompany a base-case forecast with at least one alternative scenario when uncertainty is material (new channel, macro volatility, budget cut, aggressive growth target).
- Keep the number of varied drivers small (3-5) so scenarios remain interpretable; too many simultaneously varying assumptions makes it hard to attribute the outcome to any one cause.
- Re-run scenario planning whenever a key assumption changes materially mid-period (e.g., actual CAC diverges from the base case by more than ~15-20%).
- For budget-cut scenarios, always present the efficiency-ranked option alongside the uniform cut, since decision-makers often default to "cut everything evenly" without seeing the efficiency-based alternative.

## Examples
**Best/base/worst example**: Base case assumes CAC holds at $3,000 and 400 customers acquired; best case assumes CAC improves to $2,600 (planned optimization lands) yielding 460 customers at the same budget; worst case assumes CAC rises to $3,600 due to increased competitive bidding, yielding 335 customers at the same budget — a swing of roughly ±16% in customers acquired.

**Sensitivity analysis example**: Varying CAC ±15% changes projected customer count by ±13%; varying conversion rate ±15% changes it by ±15%; varying market demand ±15% changes it by ±6%. Conversion rate and CAC are ranked as the highest-sensitivity drivers and prioritized for tighter data collection and risk mitigation.

**Budget-cut scenario example**: Facing a 20% cut, the uniform-cut option projects a 20% pipeline reduction; the efficiency-ranked cut (defunding the two lowest-ROI channels first) projects only a 9% pipeline reduction for the same dollar cut, because the defunded channels were producing disproportionately low return per dollar.

## Common Mistakes
- Presenting only a single base-case forecast with no upside/downside range when uncertainty is clearly material.
- Varying too many assumptions simultaneously across scenarios, making it impossible to tell which driver caused the outcome.
- Modeling budget cuts as a uniform percentage reduction by default, without showing an efficiency-ranked alternative.
- Failing to update scenarios when actual results diverge materially from the base case mid-period.
- Treating the "worst case" as a remote, ignorable tail risk rather than a plausible scenario worth pre-planning a response for.

## Best Practices
- Structure every material forecast as base case plus at least one alternative scenario, with the key varied drivers stated explicitly.
- Use sensitivity analysis to prioritize which 2-3 assumptions deserve the most scrutiny and data investment.
- For budget-cut situations, always present an efficiency-ranked or strategic-protection alternative alongside a uniform cut.
- Revisit scenarios on a regular cadence (at least quarterly) and immediately after any major assumption shift.

## Summary
Scenario planning converts single-point forecasts into a resilient decision tool by modeling best/base/worst outcomes and identifying, through sensitivity analysis, which assumptions matter most. For budget cuts especially, scenario planning should surface efficiency-ranked alternatives to a uniform reduction, since an evenly-applied cut is rarely the option that best protects overall return.
