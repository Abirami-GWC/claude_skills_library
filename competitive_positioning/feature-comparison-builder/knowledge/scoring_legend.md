# Scoring Legend

## Support-level scale (use consistently across the whole matrix)

| Value | Symbol | Meaning |
|---|---|---|
| Full | ✅ | Natively supported, included in standard offering |
| Partial | ◐ | Supported with meaningful limitations (scope, scale, region) |
| Add-on | ➕ | Available but requires a separate purchase/tier |
| Unsupported | ❌ | Confirmed not available |
| Unknown | ❓ | Not verified — must not be presented as ❌ |

Never substitute a different symbol set mid-table. A cell marked `Unknown`
renders as `❓` or `Not verified`, never blank — a blank cell is often misread
as "no."

## Handling nuance correctly

- "Available but as a paid add-on" → `Add-on`, detail field notes the tier/price.
- "Works, but only up to N users/GB/region" → `Partial`, detail states the limit.
- "In beta / preview, not GA" → `Partial`, detail states the release status
  (this matters most for AI Capabilities rows — see
  `references/ai_capability_comparison_notes.md`).
- "We couldn't confirm from public sources" → `Unknown` — never guess based on
  what's "probably" true for a vendor of that type.
