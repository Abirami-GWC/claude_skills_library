# Discovery-to-close playbook

Stage-based objection patterns that don't fit a single objection lookup — use this when the
question is about *where in the deal* pushback is showing up, not just *what* was said. The
`handlers/*.md` files answer "what do I say to this specific objection." This file answers "what
should I expect and prepare for at each stage," which is useful when planning a call in advance
rather than reacting to one objection in real time.

## How to use this file

Match the deal stage below, skim the objections typical to that stage, and cross-reference the
matching `handlers/` file for the actual talk track. This file is for pre-call prep and pattern
recognition, not a replacement for the handler tables.

## Stage patterns

| Stage | Typical objection shape | Root cause | Where to go for the response |
|---|---|---|---|
| Discovery | Vague deflection ("we're just looking around") | Prospect hasn't yet framed a real problem | `handlers/top-objections-playbook.md` #3, #7 |
| Demo / evaluation | Feature-gap comparisons, "does it do X like [competitor]" | Prospect is actively benchmarking | `handlers/competitor-objection-handler.md` |
| Proposal | Price, ROI, budget-cycle pushback | Cost is now concrete, so risk aversion spikes | `handlers/pricing-objection-handler.md` |
| Procurement / legal | Security review, MSA redlines, compliance asks | Deal has moved from champion to org-level scrutiny | `handlers/top-objections-playbook.md` #10 |
| Verbal-agreement stall | "Just need sign-off," goes quiet | Internal champion is under-resourced to push it through | See `negotiation-endgame-playbook.md` |

## Preparing for a call by stage

- **Discovery calls**: don't pre-load pricing or competitor talk tracks — surfacing them before
  the prospect raises them can create objections that weren't there yet. Focus on the questions in
  `handlers/top-objections-playbook.md`.
- **Evaluation-stage calls**: confirm which competitors are actually in play before the call (from
  CRM notes if available) so the right battlecard is ready rather than reasoned from scratch live.
- **Proposal-stage calls**: know the deal's approved discount range and scope-reduction options
  before the call — the pricing handler explicitly avoids answering that in the moment, so having
  it ready beforehand prevents a stall.
