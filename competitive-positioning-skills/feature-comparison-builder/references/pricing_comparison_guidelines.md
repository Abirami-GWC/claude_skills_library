# Pricing Comparison Guidelines

Pricing is the highest-stakes, fastest-changing dimension in any comparison matrix.

## Distinguish list price from negotiated/street price

Only compare **public list pricing** unless the user explicitly supplies
negotiated pricing they legitimately have. Never imply a "typical" negotiated
price for a competitor — usually confidential and varies by deal. Label
clearly: `List price (public)` vs. `Reported by prospect` vs. `Unknown`.

## Normalize the unit of comparison

Vendors price differently (per-seat, per-endpoint, per-GB, flat platform fee,
usage-based). Show the actual unit for each vendor rather than forcing a
misleading "cheaper/more expensive" verdict across incompatible units.

## Call out what's included vs. add-on

Many vendors quote a low headline price that excludes features a buyer will
likely need. Note this explicitly rather than letting the headline number stand alone.

## Freshness

Always carry the `as_of_date` for the pricing section specifically. If pricing
was fetched via a web search/fetch in this session, cite the URL. If it's from
training-data memory only, treat it as potentially stale and flag for the user
to confirm.

## Never do this

Never state a competitor's price as a specific number without a source. Never
claim "no public pricing available" without having actually checked.
