# Checklists

## Sequence Architecture Checklist
- [ ] Sequence type correctly identified
- [ ] Email count matches standard framework or stated user preference
- [ ] Each email has a distinct, non-overlapping strategic role
- [ ] Commitment level escalates logically across the sequence
- [ ] Cadence (day-based or trigger-based) is specified

## Subject Line & Preview Checklist
- [ ] Subject line under ~50 characters
- [ ] No excessive punctuation, ALL CAPS, or spam-trigger words
- [ ] Preview text extends rather than repeats the subject line
- [ ] Subject line style varies across the sequence (not all curiosity, not all direct-benefit)
- [ ] At least one A/B alternate provided for high-stakes emails

## Body Copy Checklist
- [ ] One clear framework (AIDA/PAS/StoryBrand/value-first) applied per email
- [ ] Opening line is personalized/contextual, not generic
- [ ] Paragraphs are short and mobile-skimmable
- [ ] Tone is consistent with brand/audience across all emails
- [ ] No redundant content repeated verbatim from another email in the sequence

## CTA Checklist
- [ ] Exactly one CTA per email
- [ ] CTA phrasing is specific and ownership-oriented, not generic
- [ ] CTA commitment level matches sequence position
- [ ] CTA destination is clearly stated or placeholder is marked

## Delivery Checklist
- [ ] Full sequence presented in send order
- [ ] Strategic role note included per email
- [ ] Cadence/timing summary table included
- [ ] Assumptions (audience, tone, length) stated if inferred rather than given
