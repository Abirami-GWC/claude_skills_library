# Examples

## Example 1 — Welcome Series for a Newsletter
**User Request**: "Write a 5-email welcome sequence for new subscribers to my marketing newsletter."
**Why the Skill Activates**: Explicit request for a multi-email welcome sequence.
**Workflow**: Select welcome-series architecture (5 emails/7-10 days) → gather audience/tone → draft arc → write each email.
**Expected Output**: 5 emails, each with subject/preview/body/CTA, ending in a soft CTA to explore premium content.
**Final Response**: Full 5-email sequence with strategic notes and cadence table.
**Alternative Response**: If user wants fewer emails, compress to 3 by merging story+value and social proof+CTA.
**Edge Cases**: User has no existing content to reference — ask once for their single best piece of content to feature in email 3.

## Example 2 — SaaS Trial Onboarding
**User Request**: "Create an onboarding email series for new SaaS trial users."
**Why the Skill Activates**: Named sequence type (onboarding).
**Workflow**: Select onboarding architecture, tie to usage milestones not just days.
**Expected Output**: 5-6 emails triggered by signup + product milestones.
**Final Response**: Sequence with trigger-based (not purely calendar-based) timing.
**Alternative Response**: If ESP can't support trigger-based sends, fall back to day-based cadence and note the tradeoff.
**Edge Cases**: No product details given — ask for the core feature tied to the user's stated goal before writing email 2.

## Example 3 — Cart Abandonment Flow
**User Request**: "I need a cart abandonment flow for my Shopify store."
**Why the Skill Activates**: Named sequence type with clear e-commerce context.
**Workflow**: 3-email standard architecture, 24-72 hour window.
**Expected Output**: Reminder → objection handling → incentive/urgency emails.
**Final Response**: Full 3-email flow with discount code placeholder in email 3.
**Alternative Response**: If brand doesn't discount, replace incentive email with scarcity/social proof instead.
**Edge Cases**: User doesn't want to offer a discount — substitute urgency (stock levels) or free shipping threshold.

## Example 4 — Win-back Campaign
**User Request**: "Give me a win-back campaign for customers who haven't purchased in 6 months."
**Why the Skill Activates**: Named re-engagement sequence.
**Workflow**: 3-4 email architecture, low-key open → incentive → break-up.
**Expected Output**: Full win-back sequence ending in list-cleaning CTA.
**Final Response**: Delivered with honest, non-desperate tone throughout.
**Alternative Response**: For high-value customers, add a personal outreach email option instead of automated-only.
**Edge Cases**: Brand doesn't want to lose subscribers via the break-up email — reframe last email as "stay subscribed for X" instead of unsubscribe-focused.

## Example 5 — Product Launch Sequence
**User Request**: "Draft a 7-day product launch email sequence."
**Why the Skill Activates**: Named promotional/launch sequence with explicit timeframe.
**Workflow**: 6-email launch architecture compressed into 7 days.
**Expected Output**: Teaser → reveal → proof → objection → urgency → last call.
**Final Response**: Full sequence with day-by-day send schedule.
**Alternative Response**: If launch is lower-stakes (e.g., a blog series not a paid product), compress to 3 emails.
**Edge Cases**: No firm launch date given — ask once, since urgency copy depends on a real deadline.

## Example 6 — Sequence Diagnostic/Rewrite
**User Request**: "My nurture sequence isn't converting — can you review and rewrite it?"
**Why the Skill Activates**: Diagnosing/improving an existing multi-email flow.
**Workflow**: Map each existing email to a strategic role, flag redundancy, recommend fix, rewrite.
**Expected Output**: Diagnostic table + rewritten sequence.
**Final Response**: Table showing where CTAs competed or roles duplicated, followed by revised copy.
**Alternative Response**: If the issue is subject lines only, offer a lighter-touch subject-line-only rewrite.
**Edge Cases**: User only pastes 2 of 7 emails — note the gap and offer to review the full sequence once provided.

## Example 8 — Re-engagement for a Content Newsletter
**User Request**: "I need 3 emails to win back readers who stopped opening my newsletter."
**Why the Skill Activates**: Re-engagement sequence, explicit count given.
**Workflow**: Compress standard 3-4 email win-back into requested 3 emails.
**Expected Output**: We-miss-you → what's-changed → final choice email.

## Example 10 — Ambiguous "Few Emails" Request
**User Request**: "Can you write a few emails to get people to buy my online course?"
**Why the Skill Activates**: Implies a multi-email flow even without saying "sequence."
**Workflow**: Infer this is a promotional/launch-style sequence; propose a 4-email architecture and state the assumption.
**Expected Output**: 4-email promotional sequence with a stated assumption about length/type.
**Final Response**: "I've built this as a 4-email launch sequence — let me know if you'd like it longer or shorter." followed by the sequence.
**Alternative Response**: If user clarifies it's for an evergreen (always-on) funnel rather than a launch, switch to nurture architecture instead.
**Edge Cases**: No enrollment deadline exists (evergreen course) — avoid manufactured urgency language; use social proof and curiosity instead.
