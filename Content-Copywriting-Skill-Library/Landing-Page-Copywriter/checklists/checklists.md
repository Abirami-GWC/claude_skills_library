# Checklists

## Page Architecture Checklist
- [ ] Page type (short-form/long-form/squeeze) matches offer's commitment level
- [ ] Section order follows the standard architecture for that page type
- [ ] Single primary conversion goal identified

## Hero Section Checklist
- [ ] Headline passes the 5-second clarity test
- [ ] Subheadline adds mechanism/clarification, doesn't repeat headline
- [ ] Primary CTA present and phrased specifically

## Benefits/Features Checklist
- [ ] Features are framed as visitor outcomes, not just feature lists
- [ ] Each major benefit claim has supporting proof or a flagged gap
- [ ] Specific numbers used over vague superlatives

## Social Proof Checklist
- [ ] Proof points use specific, not rounded, numbers where possible
- [ ] Testimonials/stats placed near the claims they support
- [ ] No fabricated testimonials or stats

## Objection Handling Checklist
- [ ] Top 3-5 likely objections identified
- [ ] Objections ordered from most fundamental to most specific
- [ ] No major known objection left unaddressed

## CTA Checklist
- [ ] Single consistent CTA phrase used at every placement
- [ ] Micro-copy addresses the top hesitation for this offer
- [ ] Secondary CTA (if any) clearly lower-commitment

## Delivery Checklist
- [ ] All planned sections present and in order
- [ ] Rationale note included per section
- [ ] Assumptions (page type, audience, tone) stated if inferred rather than given
- [ ] No fabricated urgency, scarcity, or proof
