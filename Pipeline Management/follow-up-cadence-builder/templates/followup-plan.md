# Follow-Up Plan Template

Present the cadence as a dated schedule, followed by drafted messages for each
touch, and an explicit stopping point. Don't bury the schedule under prose —
lead with the dates.

## Standard layout

```
## Follow-Up Plan — [Deal Name]

**Current situation:** [1-2 sentence summary: stage, last contact, what happened]

| # | Date | Channel | Message Type | Purpose |
|---|---|---|---|---|
| 1 | Jul 16 | Email | Proposal follow-up | Check in on proposal, offer a call |
| 2 | Jul 19 | Call | Proposal follow-up | Direct ask about timeline if no reply |
| 3 | Jul 22 | Email | Break-up | Ask whether to close out the opportunity |

**Stopping rule:** If touch #3 goes unanswered, stop this cadence and flag the deal
for a health/risk review rather than continuing to follow up automatically.

---

### Drafted messages

[One drafted message per touch that involves sending something, using
templates/email-template.md's message-type patterns. Call touches get a short
talking-points list instead of a full script.]

### Reminders

[One line per touch, using templates/reminder-template.md, suitable for pasting
into a task manager.]
```

## Tone

Scannable first, detailed second — a rep should be able to glance at the table and
know exactly what happens next and when, then scroll down for the actual message
text only when they're ready to send it.
