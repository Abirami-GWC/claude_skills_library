# Risk Analysis Template

Only include rows for risks that are actually present — don't pad this table with
generic sales risk boilerplate that doesn't apply to this specific pipeline.

## Layout

```
| Reason | Impact | Recommended Action |
|---|---|---|
| [Deal name] is [X]% of the total forecast | If this deal slips, the forecast drops by $[amount] | Prioritize confirming this deal this week; have a fallback plan if it slips |
| [Deal name] has had no activity in [N] days despite being [tier] confidence | Confidence may be overstated | Re-engage before counting on this deal |
| [N] deals have no stated close date | Their timing within this period is unconfirmed | Get updated close-date estimates before finalizing the forecast |
```

## If nothing significant is flagged

Say so plainly rather than omitting the section or inventing a risk:

```
No significant risks identified — the forecast isn't overly dependent on any
single deal, and all included deals have recent activity.
```
