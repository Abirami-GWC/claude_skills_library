# Examples

## Example 1 — SaaS Product Landing Page
**User Request**: "Write landing page copy for my new project management SaaS tool."
**Why the Skill Activates**: Explicit landing page copy request for a product.
**Workflow**: Select short-form SaaS architecture → hero → benefits → social proof → FAQ → final CTA.
**Expected Output**: Full page copy with consistent "Start my free trial" CTA throughout.
**Final Response**: Section-by-section copy with rationale notes.
**Alternative Response**: If pricing is complex/high-price, shift toward a longer architecture with more objection handling.
**Edge Cases**: No testimonials available — flag the gap and mark placeholder testimonial slots.

## Example 2 — High-Ticket Course Sales Page
**User Request**: "I need a sales page for my $2,000 online course."
**Why the Skill Activates**: Sales page request for a high-commitment offer.
**Workflow**: Select long-form sales page architecture — problem agitation, story, detailed benefits, multiple proof points, guarantee, pricing, urgency if genuine.
**Expected Output**: Long-form page copy with heavy proof and objection handling before the price reveal.
**Final Response**: Full page with a dedicated guarantee/risk-reversal section.
**Alternative Response**: If no real deadline exists, remove urgency language and rely on social proof/guarantee instead.
**Edge Cases**: User wants fake urgency ("only 3 spots left" when untrue) — decline to fabricate false scarcity and suggest genuine alternatives (bonus expiring, cohort start date).

## Example 3 — Webinar Registration (Squeeze) Page
**User Request**: "Write copy for a webinar registration page."
**Why the Skill Activates**: Lead-gen page with single conversion action.
**Workflow**: Select squeeze-page architecture — hero, 3-5 benefit bullets, single opt-in form, minimal FAQ.
**Expected Output**: Short, focused page copy.
**Final Response**: Hero + bullets + CTA + trust/privacy note.
**Alternative Response**: If webinar has a strong specific instructor credibility angle, add a brief one-line credibility note near the hero.
**Edge Cases**: User wants a long, feature-heavy page for a free webinar — recommend trimming since over-persuading a low-commitment offer can suppress conversion.

## Example 4 — Landing Page Audit
**User Request**: "Review my landing page and tell me why it's not converting."
**Why the Skill Activates**: Diagnosing/improving an existing landing page.
**Workflow**: Map existing sections to standard architecture, flag missing proof/objection handling/CTA inconsistency.
**Expected Output**: Diagnostic table + prioritized fix recommendations.
**Final Response**: Table (section/issue/fix) followed by top 3 priority changes.
**Alternative Response**: If the user only wants the hero reviewed, scope the audit to that section alone.
**Edge Cases**: User pastes only partial page content — note the gap and offer to complete the audit once the full page is provided.

## Example 5 — Product Page with Just Hero + FAQ Requested
**User Request**: "Can you write the hero section and FAQ for our product page?"
**Why the Skill Activates**: Section-level request within the context of a landing page.
**Workflow**: Write hero (headline/subheadline/CTA) and FAQ (top objections as Q&A), without generating unrequested sections.
**Expected Output**: Hero copy + FAQ block only.
**Final Response**: Two sections delivered, with a note offering to complete the rest of the page if useful.
**Alternative Response**: If audience/objections aren't given, ask once for the top 2-3 customer questions before writing the FAQ.
**Edge Cases**: No product details for FAQ — ask for at least the core offer before writing objection content.

## Example 6 — E-commerce Product Landing Page
**User Request**: "Write a product landing page for my new water bottle brand."
**Why the Skill Activates**: Product-focused conversion page request.
**Workflow**: Short-form architecture — hero, benefits (durability, design, sustainability), social proof, FAQ (shipping/returns), final CTA.
**Expected Output**: Full page with e-commerce-specific objection handling (shipping, returns, materials).
**Final Response**: Section-by-section copy with a consistent "Shop now" or "Get yours" CTA.
**Alternative Response**: If targeting a gift-buying audience, adjust hero framing toward gifting occasion.
**Edge Cases**: No differentiators provided vs. competitors — ask once, since differentiation drives the core benefit framing.

## Example 7 — Free Tool / Lead Magnet Landing Page
**User Request**: "I need a landing page for a free ROI calculator we built to generate leads."
**Why the Skill Activates**: Lead-gen page with single opt-in/download conversion action.
**Workflow**: Squeeze-page architecture, benefit-driven headline, minimal friction.
**Expected Output**: Short, focused page with a single CTA to access the tool.
**Final Response**: Hero + 3 benefit bullets + CTA + privacy trust note.
**Alternative Response**: If lead-gen goal is secondary to actual tool usage, shift framing toward tool value first, email capture second.
**Edge Cases**: Legal requires specific privacy disclosure language — include a placeholder and note that legal review is required.

## Example 8 — B2B Demo Request Page
**User Request**: "Write landing page copy to get enterprise prospects to book a demo."
**Why the Skill Activates**: Conversion-focused page with a clear B2B goal (demo request).
**Workflow**: Mid-length architecture — hero, ROI-focused benefits, case study/logo social proof, FAQ (security, implementation), CTA "Book a demo."
**Expected Output**: Full page with enterprise-appropriate proof (logos, case studies) and objection handling (security/compliance).
**Final Response**: Section-by-section copy with consistent "Book a demo" CTA.
**Alternative Response**: If a self-serve trial also exists, add it as a clearly secondary, lower-commitment CTA.
**Edge Cases**: No case studies available yet (new product) — substitute founder credibility or beta-user quotes, clearly framed as early-stage proof.

## Example 9 — Nonprofit Donation Landing Page
**User Request**: "Write a landing page to get people to donate to our nonprofit's campaign."
**Why the Skill Activates**: Conversion page with a specific action (donate).
**Workflow**: Mid-length architecture — emotionally resonant hero, impact-framed benefits ("your $50 provides X"), social proof (impact stats, testimonials), FAQ (where funds go), CTA "Donate now."
**Expected Output**: Full page emphasizing tangible impact per contribution level.
**Final Response**: Section-by-section copy with transparency-focused objection handling.
**Alternative Response**: If campaign has a matching-gift deadline, add genuine urgency framing tied to that deadline.
**Edge Cases**: No specific impact metrics available — ask once, since donation pages convert significantly better with concrete impact framing than vague appeals.

## Example 10 — Ambiguous "Website Copy" Request
**User Request**: "Can you help me write copy for my homepage? I want more people to sign up for a free trial."
**Why the Skill Activates**: Not phrased as "landing page," but the goal (drive a specific conversion action) and structure needed match this skill.
**Workflow**: Treat as a short-form SaaS landing page; confirm/state the assumption that the homepage is functioning as the primary conversion page.
**Expected Output**: Full page copy structured as a conversion-focused homepage.
**Final Response**: "I've approached this as your primary conversion page for free trial signups — let me know if the homepage serves other purposes too." followed by full copy.
**Alternative Response**: If the homepage also needs to serve non-conversion visitors (e.g., investors, press), note that a dedicated landing page for the trial CTA may convert better than overloading the homepage.
**Edge Cases**: User wants the homepage to serve multiple audiences/goals — recommend a dedicated landing page for the trial CTA rather than diluting the homepage's single conversion focus.
