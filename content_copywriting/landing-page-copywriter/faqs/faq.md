# FAQ

1. **What's the difference between a landing page and a homepage?** A landing page is built around one conversion goal; a homepage often serves multiple audiences and goals. This skill treats any page built around a single conversion action the same way, homepage or not.
2. **How long should a landing page be?** It depends on the offer's commitment level — see `references/landing_page_frameworks.md`; low-commitment offers need short pages, high-commitment offers need longer, proof-heavy pages.
3. **Should the CTA wording change throughout the page?** No — use one consistent CTA phrase at every placement.
4. **What if I don't have testimonials yet?** The skill will flag the gap and either omit the social proof section or mark clearly-labeled placeholders.
5. **Can this skill write just the hero section?** Yes, section-level requests are supported.
6. **How is this different from Ad Copy Generator?** Ad Copy Generator writes paid ad copy (headlines/hooks for ad platforms); this skill writes the destination page the ad sends traffic to.
7. **How is this different from Email Sequence Writer?** Email Sequence Writer builds multi-email owned-channel sequences; this skill builds a single conversion-focused web page.
8. **Does the skill guarantee a certain conversion rate?** No — it applies established conversion-copywriting principles but cannot guarantee performance without live data.
9. **Can it write actual HTML/CSS for the page?** No, it produces organized copy by section; implementation is out of scope.
10. **What if my offer has multiple possible conversion goals (e.g., both "buy now" and "book a demo")?** The skill will recommend one primary CTA and treat any other action as a clearly secondary, lower-commitment CTA.
11. **Should urgency/scarcity always be used?** Only when genuine; the skill avoids fabricating false urgency.
12. **How many objections should the FAQ address?** Typically the top 3-5 most common objections for that offer/audience.
13. **What if I want a very short, minimal landing page?** The squeeze-page architecture is designed exactly for this.
14. **Can the skill review a competitor's landing page for inspiration?** It can analyze structure/approach if the content is provided, but should not reproduce competitor copy verbatim.
15. **Does the skill handle pricing page copy specifically?** Yes, pricing is treated as one section within the larger page architecture, especially in long-form sales pages.
16. **What if my product has no clear differentiator yet?** The skill will ask once, since differentiation drives the core benefit framing.
17. **Can I get multiple headline options?** Yes, 2-3 headline variants are typically provided for the hero section.
18. **Should social proof be one big block or spread throughout?** Spread near the specific claims each proof point supports, not only in one generic testimonials block.
19. **What if the page needs legal/compliance disclosures?** The skill can include placeholder disclosure language but does not provide legal review.
20. **How does the skill decide page architecture?** Based on the offer's price/commitment level and conversion goal — see `references/landing_page_frameworks.md`.
21. **Can this skill be used for app store listing copy?** The core principles (clarity, benefits, social proof) apply, but app store character limits require adapted, more compressed copy.
22. **What if I already have a page and just want the FAQ section improved?** The skill can scope down to just that section using the FAQ-specific prompt.
23. **Should every landing page have a guarantee?** Only if the brand genuinely offers one; a guarantee is recommended especially for higher-price offers to reduce perceived risk.
24. **How does this skill avoid generic, fluffy copy?** By requiring proof next to every benefit claim and specific rather than vague language throughout.
25. **Can the skill suggest SEO-friendly headlines?** As a bonus it can suggest keyword-aware phrasing, but full technical SEO optimization is out of scope.
