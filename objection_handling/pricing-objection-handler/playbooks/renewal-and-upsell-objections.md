# Renewal and upsell objections

Objections from existing customers at renewal or expansion time. These differ from new-logo
objections in `handlers/top-objections-playbook.md` because there's already a usage history and a
relationship to draw on — use that context instead of a cold-deal response frame.

## How to use this file

Match the objection, then check "what to pull first" before responding — renewal objections are
usually answerable more credibly with the account's own data than with a generic talk track.

## Renewal and expansion objections

| # | Objection | What to pull first | Response frame |
|---|---|---|---|
| 1 | "We're not seeing the ROI" | Usage/adoption data for the account | Show actual usage against the goals set at onboarding; if usage is genuinely low, address adoption before defending renewal price. |
| 2 | "We want to shop around this time" | Renewal date, contract terms, satisfaction history (support tickets, NPS/CSAT if logged) | Don't get defensive — ask what's prompting the re-evaluation now; it's often a new stakeholder or a budget event, not dissatisfaction. |
| 3 | "Can we downgrade / reduce seats?" | Current utilization by seat/module | Ask what's changed in their usage pattern before agreeing; sometimes a downgrade request is really a pricing objection in disguise — check against `handlers/pricing-objection-handler.md`. |
| 4 | "We'd renew but not at this price increase" | The account's pricing history and the increase rationale (if one exists) | Route to the approved renewal-pricing owner per the hard rule below — same restriction as new-deal pricing. |
| 5 | "We want to add [feature/module] but not sure it's worth the upsell cost" | Usage data that predicts the new module's value for this specific account | Anchor the upsell pitch to a workflow they already run, not a generic feature list. |

## Hard rule

Same restriction as `handlers/pricing-objection-handler.md`: this playbook never states a specific
renewal discount, credit, or concession. Route pricing flexibility questions to the approved
renewal-pricing owner.

## Difference from new-logo handling

A renewal conversation has a paper trail — use it. Before reaching for a generic response frame,
check whether the account has:
- Usage/adoption data (often the single best rebuttal to "not seeing ROI")
- A support or escalation history (explains a "we're frustrated" objection instantly)
- A named champion who may have changed roles (a common, non-obvious cause of "we're
  re-evaluating")
