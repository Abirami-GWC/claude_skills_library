# Post Structure Frameworks

## 1. Hook → Tension → Resolution → CTA
Best for: personal stories, case studies, "what we learned" posts.
- Hook: drop reader into a moment or a stakes-laden statement.
- Tension: what was hard, uncertain, or at risk. Don't resolve too fast — let it breathe for 2-3 short lines.
- Resolution: what happened, what changed, what was learned. Be specific (numbers, concrete detail).
- CTA: invite reflection, a question, or a related ask.

## 2. Hook → Claim → Evidence → CTA
Best for: opinion posts, thought leadership, contrarian takes.
- Hook: state the take plainly, no hedging.
- Claim: expand the argument in 2-3 sentences.
- Evidence: one concrete example, data point, or experience that backs it up. One is enough — don't over-stuff.
- CTA: invite agreement/disagreement, or a related question.

## 3. Hook → Problem → Framework/Steps → CTA
Best for: educational, how-to, "here's our process" posts.
- Hook: name the problem or promise the payoff.
- Problem: why this is harder or more common than people think (1-2 lines).
- Framework/Steps: the actual value — numbered or clearly delineated steps/principles. Keep each step to 1-2 lines; a carousel format can expand each into its own slide.
- CTA: save/share prompt, or offer to go deeper (e.g. "template in the comments").

## 4. Hook → Before/After → Mechanism → CTA
Best for: transformations, case studies, product results.
- Hook: state the outcome/result up front.
- Before/After: contrast starting point and result concretely.
- Mechanism: the specific thing that caused the change — this is the credibility layer, don't skip it.
- CTA: relate it back to the reader ("if you're seeing X, this might be why").

## Choosing a framework
- Story/personal content → Framework 1
- Opinion/take content → Framework 2
- Tutorial/process content → Framework 3
- Results/case-study content → Framework 4

If the input doesn't cleanly fit one framework, default to whichever framework matches the goal: engagement/discussion → Framework 2; saves/shares → Framework 3; trust-building → Framework 1 or 4.
