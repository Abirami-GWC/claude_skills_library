# Topic → Angle Mapping

Use this to decide which angles are worth generating for a given input type.

| Input type | Strong angles | Weak/avoid angles |
|---|---|---|
| Product launch / feature update | Behind-the-scenes ("why we built this"), direct value (how it solves X), data-led (early results) | Generic announcement ("Excited to share...") |
| Blog post / long-form content | Curiosity gap (one surprising insight pulled out), how-to (a sub-piece of the framework), contrarian (the one pushback-worthy claim in it) | Trying to summarize the whole post in one caption |
| Company milestone / funding / award | Confession/vulnerability (what it took, what almost didn't work), story open | Pure celebration post with no substance |
| Customer story / case study | Data-led (the specific number), story open (the moment things changed for them) | Vague "our customers love us" framing |
| Industry news / trend | Contrarian (your actual take, not a recap), question hook (if genuinely undecided) | Neutral news recap with no opinion |
| Personal lesson / failure | Confession, story open | Over-polished "lessons learned" listicle that hides the real story |
| Data / research finding | Data-led, curiosity gap | Burying the number three lines in |
| Hiring / team update | Behind-the-scenes, direct value (what it's like to work there) | Generic "we're hiring!" with no texture |

## How to use
1. Identify which row the input best matches (an input can match more than one).
2. Pull 2-3 angles from the "strong" column per platform requested.
3. Skip the "weak" angles entirely unless the user specifically asks for a straightforward announcement style.
