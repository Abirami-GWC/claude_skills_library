# Discovery Checklist (use when the input pain points are vague or incomplete)

- [ ] Is the pain described specifically enough to map to a feature? (e.g. "things are slow" needs more detail than "VPN disconnects")
- [ ] Do we know the customer's industry/segment? (helps select a relevant case study)
- [ ] Do we know the audience for the demo? (executive / technical / POC)
- [ ] Do we know which pain the customer considers most urgent?
- [ ] Are there likely objections we should prepare for? (check mappings/objection_mapping.yaml)
- [ ] If critical info is missing, ask ONE targeted clarifying question rather than guessing on multiple fronts at once
