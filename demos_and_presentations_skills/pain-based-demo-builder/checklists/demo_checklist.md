# Demo Checklist (self-check before finishing)

- [ ] Business challenge explained in the customer's own words
- [ ] Every agenda item is phrased as a solved problem, not a feature name
- [ ] Agenda is sequenced: most urgent -> quick win -> foundational -> advanced -> proof -> Q&A
- [ ] No more than ~8 agenda items (related pains grouped where needed)
- [ ] Case study (if included) pulled from knowledge/case_studies.md — nothing fabricated
- [ ] Any stat or metric used traces back to knowledge/success_metrics.md
- [ ] If the user requested a specific audience (executive/technical/POC), the matching template was used
- [ ] Ends with Q&A (or, for POC, an exit-criteria review)
- [ ] Any pain not found in mappings/pain_to_feature.yaml was flagged, not silently invented
- [ ] Agenda opens with "Understanding Your Current Challenges" and closes with "Next Steps"
- [ ] If an industry was stated, the matching playbooks/industries/ file (if one exists) was used to weight/frame the agenda
- [ ] If an incumbent vendor was mentioned, the competitive_vendor angle from objection_mapping.yaml was considered (not a head-to-head comparison)
- [ ] If company size or site count was mentioned, sizing_guidance.md was checked — any inferred-but-unstated pain (e.g. multi-site management) was offered as a suggestion, not stated as if the customer said it
