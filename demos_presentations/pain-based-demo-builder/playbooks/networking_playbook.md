# Networking Playbook

## Pain: VPN disconnects / unreliable remote access

**Demo Flow**
1. Show VPN stability / setup wizard
2. Show branch office / multi-site connectivity
3. Show monitoring dashboard for connection health
4. Show automatic failover

**Business Value**
Reduced downtime; fewer help desk tickets; consistent remote worker experience.

## Pain: Poor network performance / slow WiFi

**Demo Flow**
1. Show network performance monitoring
2. Show traffic shaping / QoS controls
3. Show capacity planning / usage reporting

**Business Value**
Improved user experience; data-driven capacity decisions.

## Pain: Difficult to manage multiple sites

**Demo Flow**
1. Show centralized multi-site management console
2. Show templated site deployment
3. Show unified policy management across sites

**Business Value**
Reduced IT overhead; faster new-site rollout; consistent policy enforcement.

## Sequencing note
Fix connectivity/reliability first — it's foundational and usually the most
emotionally charged pain — before moving to performance optimization and
multi-site management, which are "nice to have it working well" pains.
